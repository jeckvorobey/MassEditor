## MODIFIED Requirements

### Requirement: Пакетная загрузка связанных данных
Список товаров MUST загружать категории, теги, SKU и остатки батчами, без N+1 запросов по каждому товару. При выбранном фильтре склада отображаемый остаток MUST относиться только к этому складу; без выбранного склада MUST сохраняться детализация по всем складам.

#### Scenario: Остатки по всем складам без фильтра
- **WHEN** страница содержит товары и фильтр `stock_id = 0`
- **THEN** `attachStockDetails()` MUST одним батчем загрузить остатки по product IDs, сохранить общий `count` товара и показать `∞`, если хотя бы один складской остаток SKU бесконечный; `stock_details` MUST содержать детали всех складов, участвующих в учете товара

#### Scenario: Остаток выбранного склада
- **WHEN** страница содержит товары и фильтр `stock_id > 0`
- **THEN** сервис MUST одним батчем ограничить агрегацию остатков этим складом, установить отображаемый `count` товара по выбранному складу и вернуть не более одной записи в `stock_details` для выбранного склада

#### Scenario: Нулевой или бесконечный остаток выбранного склада
- **WHEN** у товара есть складская запись для выбранного склада с `count = 0` или хотя бы один SKU имеет `count IS NULL`
- **THEN** сервис MUST вернуть соответственно `0` или `null`, а backend UI MUST показать `0` или `∞` без подмены общим остатком товара

#### Scenario: Скрытие детализации выбранного склада
- **WHEN** backend UI отображает строку товара при `filters.stock_id > 0`
- **THEN** desktop- и mobile-разметка MUST показать остаток выбранного склада и MUST НЕ рендерить кнопку `Склады` и popover с перечнем складов

#### Scenario: Кнопка складов без фильтра
- **WHEN** backend UI отображает строку товара при `filters.stock_id = 0` и у товара есть складские детали
- **THEN** desktop- и mobile-разметка MUST сохранить кнопку `Склады` и popover с деталями всех складов
