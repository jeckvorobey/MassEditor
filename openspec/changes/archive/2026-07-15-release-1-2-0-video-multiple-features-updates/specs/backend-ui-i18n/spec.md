## ADDED Requirements

### Requirement: UI видео и множественных характеристик
Backend UI MUST показывать только поля и режимы, допустимые для выбранной операции и типа характеристики, и MUST передавать серверу нормализуемые значения без обхода общего подтверждения.

#### Scenario: Форма видео
- **WHEN** пользователь выбирает операцию `video`
- **THEN** UI MUST показать режимы `set`/`clear`, поле URL только для `set` и локализованную клиентскую ошибку для неверного URL

#### Scenario: Форма множественной характеристики
- **WHEN** пользователь выбирает характеристику с `multiple=1`
- **THEN** UI MUST показать режимы `replace`, `add`, `remove`, `clear` и multi-select существующих значений этой характеристики

#### Scenario: Очистка без значений
- **WHEN** выбран режим `clear` для видео или характеристики
- **THEN** UI MUST скрыть и отключить поле значения, чтобы устаревший ввод не попал в POST

#### Scenario: Подтверждение
- **WHEN** обязательные поля валидны и пользователь открывает modal
- **THEN** modal MUST показать локализованные название операции, режим и число товаров, а submit MUST добавить `confirm_apply=1`

### Requirement: Локализация релиза 1.2.0
Все новые видимые строки видео и множественных характеристик MUST поступать из `shopMasseditorPluginI18nService` и JS i18n dictionary и MUST иметь значения в `ru_RU` и `en_US`.

#### Scenario: Русский интерфейс
- **WHEN** `interface_language=ru_RU`
- **THEN** подписи режимов, URL, multi-select, подтверждение и ошибки MUST отображаться по-русски

#### Scenario: Английский интерфейс
- **WHEN** `interface_language=en_US`
- **THEN** подписи режимов, URL, multi-select, подтверждение и ошибки MUST отображаться по-английски без русского fallback

#### Scenario: Экранирование
- **WHEN** UI отображает видеоссылку, название характеристики, значение или серверную ошибку
- **THEN** Smarty/JS MUST использовать безопасный текстовый вывод без интерпретации пользовательского HTML
