## Why

MassEditor уже покрывает основные массовые операции, но для быстрого выпуска следующей коммерческой версии ему не хватает массового управления видео, безопасной работы с множественными значениями характеристик и штатного пути обновления схемы установленного плагина. Релиз `1.2.0` закрывает эти три пробела без расширения в SKU-характеристики, дочерние характеристики и отдельную систему миграций.

## What Changes

- Добавить массовую операцию видео с режимами установки одной ссылки всем выбранным товарам и очистки ссылки.
- Сохранять видео через документированный объект `shopProduct` и отклонять неподдерживаемую ссылку до изменения товаров.
- Разрешить множественные характеристики товара и режимы `replace`, `add`, `remove`, `clear`.
- Гарантировать, что операция характеристики меняет только выбранную характеристику уровня товара, не затрагивает строки SKU, другие характеристики и не использует значения другой характеристики.
- Перевести сохранение характеристик на штатный документированный контракт данных товара Shop-Script с массивом значений вместо ручной записи связей в таблицы Shop-Script.
- Добавить идемпотентный meta-update в `lib/updates/` и полагаться на штатную последовательность Webasyst через `wa_app_settings.update_time`; собственная таблица миграций не создаётся.
- Повысить версию плагина до `1.2.0` и удалить оставшееся текущее упоминание отменённого идентификатора без миграции с него.
- Добавить русские и английские тексты, подтверждение, журналирование и TDD-покрытие новых сценариев.

Не входят в релиз: отдельные видео для каждого товара, загрузка видеофайлов, характеристики SKU/модификаций, дочерние характеристики, создание новых справочных значений пользователем, marketplace API, экспорт журнала и собственная таблица `migrations`.

Перед реализацией используются официальные источники Webasyst/Shop-Script:

- https://developers.webasyst.ru/docs/cookbook/meta-updates/
- https://developers.webasyst.ru/apps/shop-script/shopProduct/
- https://developers.webasyst.ru/api/explorer/shop/shop.product.add/
- https://developers.webasyst.ru/docs/cookbook/plugins/
- https://developers.webasyst.ru/docs/cookbook/basics/file-structure/

## Capabilities

### New Capabilities

Новых изолированных capability не создаётся: функции расширяют существующие контракты плагина.

### Modified Capabilities

- `mass-operations`: добавить whitelisted-операцию видео и безопасные режимы множественных характеристик.
- `backend-ui-i18n`: добавить элементы управления, подтверждение и полный словарь `ru_RU`/`en_US` для видео и множественных характеристик.
- `plugin-architecture`: зафиксировать версию `1.2.0` и штатные идемпотентные meta-updates Webasyst без собственной таблицы миграций.
- `quality-workflow`: добавить TDD и проверки повторного запуска meta-update, транзакционной безопасности и официальных контрактов Shop-Script.

## Impact

- Backend: `lib/actions/`, `lib/classes/`, `templates/actions/backend/Backend.html`, `js/masseditor.js`, i18n-слой и PO-файлы.
- Данные Shop-Script: `shop_product.video_url` и характеристики товара через `shopProduct`; прямые широкие удаления из `shop_product_features` исключаются.
- Собственная БД плагина: только идемпотентная проверка/доведение текущей схемы `shop_masseditor_log`; новых таблиц не требуется.
- Релиз: `lib/config/plugin.php`, `lib/updates/`, Docker bootstrap cleanup, тесты и материалы версии `1.2.0`.
- Внешние зависимости и storefront-код не добавляются.
