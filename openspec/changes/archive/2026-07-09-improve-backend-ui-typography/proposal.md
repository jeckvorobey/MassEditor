## Why

В backend UI MassEditor ключевые области интерфейса используют слишком мелкую типографику: заголовки таблиц, табличные значения, подписи, кнопки и мобильные карточки товара читаются тяжело и выглядят визуально сжатыми. На мобильных экранах проблема усиливается: текст становится мелким, а интерактивные элементы получаются неудобными для нажатия.

Задачу нужно закрыть сейчас, потому что она напрямую влияет на повседневную работу с массовым редактированием товаров и снижает удобство backend-интерфейса без изменения бизнес-логики плагина.

## What Changes

- Увеличить базовую типографическую шкалу backend UI для desktop и mobile.
- Повысить читаемость таблиц и мобильных карточек товаров: значения, подписи, бейджи, пагинация, метаданные.
- Увеличить размеры интерактивных элементов backend UI: полей ввода, select, кнопок, элементов выбора и мобильной панели применения.
- Сохранить текущую структуру backend-экрана, backend route `?plugin=masseditor`, локализацию и существующую логику операций без расширения scope.
- Зафиксировать контракт читаемости и touch targets тестами и OpenSpec-дельтой.

## Capabilities

### New Capabilities

- Нет.

### Modified Capabilities

- `backend-ui-i18n`: backend UI должен обеспечивать читаемую типографику и удобные интерактивные размеры на desktop и mobile без изменения текущего пользовательского потока.

## Impact

- Затронутые файлы: `wa-apps/shop/plugins/masseditor/css/masseditor.css`, `tests/js/masseditor.test.js`.
- Возможна точечная сверка `templates/actions/backend/Backend.html`, если для touch targets понадобятся дополнительные CSS hooks без изменения текста или маршрутов.
- API, бизнес-логика массовых операций, PHP-классы, модели и локализационные каталоги не меняются.
- Перед реализацией проверяются официальные источники Webasyst по структуре и подключению frontend-ассетов плагина, а также актуальные веб-рекомендации по читаемой типографике и touch target size.
