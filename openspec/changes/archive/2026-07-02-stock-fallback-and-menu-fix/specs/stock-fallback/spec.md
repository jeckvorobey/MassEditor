## ADDED Requirements

### REQ-STOCK-FALLBACK-1: Fallback на SKU count при отсутствии складского учёта

Когда массовая операция «Остатки» выполняется с указанием склада (`stock_id > 0`), но товар не имеет складских остатков в `shop_product_stocks` (т.е. не использует складской учёт), система должна обновлять `count` напрямую в `shop_product_skus`, а не пытаться записать в `shop_product_stocks`.

**Acceptance criteria:**
- Если ни один SKU товара не имеет записей в `stock`, операция проводится через `sku.count`
- Если хотя бы один SKU имеет складские остатки, используется складская операция
- Значение применяется корректно (set/increase/decrease)
- Операция записывается в журнал

### REQ-STOCK-FALLBACK-2: Определение складского учёта

Метод `usesWarehouseStockAccounting()` определяет, использует ли товар складской учёт, проверяя наличие непустого массива `stock` в SKU.

**Acceptance criteria:**
- Возвращает `true`, если хотя бы один SKU имеет `stock` как непустой массив
- Возвращает `false`, если ни один SKU не имеет `stock`
- Не выполняет дополнительных SQL-запросов
