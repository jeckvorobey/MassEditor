## ADDED Requirements

### Requirement: Фильтр по типу учёта остатков
При массовой операции «Остатки» без выбора склада (`stock_id = 0`) UI MUST позволять выбрать один из режимов: «Все товары», «Без складского учёта» или «Со складским учётом».

#### Scenario: Отображение без выбранного склада
- **WHEN** пользователь выбирает операцию «Остатки» и `stock_id = 0`
- **THEN** UI MUST показать фильтр по типу учёта остатков

#### Scenario: Конкретный склад выбран
- **WHEN** пользователь выбирает конкретный склад с `stock_id > 0`
- **THEN** UI MUST скрыть фильтр по типу учёта остатков

#### Scenario: Значение по умолчанию
- **WHEN** фильтр открывается без ранее сохранённого значения
- **THEN** UI MUST выбрать «Все товары» для сохранения обратной совместимости

#### Scenario: Payload операции
- **WHEN** пользователь выбирает значение фильтра и отправляет операцию
- **THEN** выбранное значение MUST передаваться как `stock_type_filter`

### Requirement: Применение фильтра на backend
Перед применением операции «Остатки» backend MUST фильтровать выбранные товары по значению `stock_type_filter`.

#### Scenario: Все товары
- **WHEN** `stock_type_filter=all`
- **THEN** backend MUST сохранить текущее поведение и рассмотреть все выбранные товары

#### Scenario: Только товары без складского учёта
- **WHEN** `stock_type_filter=without_warehouse`
- **THEN** backend MUST применять операцию только к товарам, у которых `usesWarehouseStockAccounting() === false`

#### Scenario: Только товары со складским учётом
- **WHEN** `stock_type_filter=with_warehouse`
- **THEN** backend MUST применять операцию только к товарам, у которых `usesWarehouseStockAccounting() === true`

#### Scenario: Журнал пропущенных товаров
- **WHEN** товар исключён выбранным фильтром
- **THEN** backend MUST записать пропуск товара в журнал операции с указанием причины
