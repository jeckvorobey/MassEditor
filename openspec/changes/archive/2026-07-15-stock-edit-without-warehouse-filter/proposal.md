## Why

Сейчас массовая операция «Остатки» без выбора склада (`stock_id = 0`) обновляет `count` в `shop_product_skus` для всех выбранных товаров одинаково. Но в каталоге могут быть товары с складским учётом (есть записи в `shop_product_stocks`) и без него. Единый подход не учитывает эту разницу — товары со складским учётом теряют корректность данных, а товары без складского учёта обрабатываются нормально.

Нужно исследовать поведение и добавить фильтр, который позволит разделять товары по типу учёта остатков при массовой операции без указания склада.

## What Changes

- **Исследование**: как Shop-Script ведёт себя при `stock_id = 0` для товаров с/без складского учёта, какие побочные эффекты возможны
- **Фильтр по типу учёта остатков**: добавить в UI фильтр, позволяющий выбрать — применять операцию только к товарам без складского учёта, только к товарам со складским учётом, или ко всем
- **Логика применения**: при `stock_id = 0` и фильтре «без складского учёта» — обновлять `sku.count`; при фильтре «со складским учётом» — пропускать или обрабатывать отдельно

## Capabilities

### New Capabilities
- `stock-type-filter`: Фильтрация товаров по типу учёта остатков (складской / безскладской / все) при массовой операции «Остатки» без выбора склада.

### Modified Capabilities
- `stock-fallback`: Расширить логику fallback — учитывать фильтр типа учёта при определении способа применения операции.

## Impact

- Backend: `shopMasseditorPluginMassOperationService` — новая логика фильтрации перед применением операции
- Frontend: `masseditor.js` — UI-элемент фильтра в блоке операции «Остатки»
- Шаблоны: possibly `templates/` — добавление select/radio для фильтра
- Тесты: новые тест-кейсы для каждого сценария фильтра
- Локализация: RU/EN строки для фильтра
