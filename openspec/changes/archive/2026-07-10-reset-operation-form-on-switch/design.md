## Context

`setOperation()` переключает видимость и disabled-состояние групп с `data-operation-fields`, но оставляет их текущие значения. Операции `price` и `compare_price` имеют общие поля, поэтому сброс должен выполняться по факту смены операции, а не только для скрытых групп.

## Goals / Non-Goals

**Goals:**

- При выборе другой операции очищать text/textarea и возвращать select к первому варианту во всех полях операций.
- Сохранять выбранные товары, фильтры и другие неоперационные данные.
- Не очищать форму при повторном выборе уже активной операции.

**Non-Goals:**

- Не изменять серверную обработку, CSRF, БД, локализацию или список операций.
- Не менять значения, введенные пользователем, при изменении mode внутри той же операции.

## Decisions

1. Перед `setOperation()` при клике сравнивать предыдущую и запрошенную операции. При отличии сбрасывать все controls внутри `[data-operation-fields]`.

   Альтернатива: сбрасывать только скрываемые группы. Отклонена: общие поля `price`/`compare_price` останутся с предыдущими значениями.

2. Для `input` и `textarea` устанавливать пустую строку, для `select` — первый вариант; после этого повторно выполнить существующие функции видимости.

   Альтернатива: вызвать `form.reset()`. Отклонена: он затронет выбор товаров, фильтры и скрытые служебные поля.

## Risks / Trade-offs

- [Risk] Сброс выполнится при повторном выборе активной операции. → Mitigation: сравнивать ID операции до сброса.
- [Risk] Сброс режима склада оставит ошибочную подсветку. → Mitigation: очищать состояние ошибки выбранного склада при смене операции.
