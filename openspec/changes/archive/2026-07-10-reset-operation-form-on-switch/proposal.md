## Why

При выборе другой операции значения из предыдущей формы остаются заполненными. Пользователь может применить неподходящее значение к новой операции, поэтому переход должен начинаться с чистой формы новой операции.

Перед реализацией сверяется официальная [документация Webasyst по плагинам](https://developers.webasyst.ru/docs/cookbook/plugins/): изменение остается изолированным в JS backend UI плагина и не затрагивает ядро Shop-Script.

## What Changes

- При фактической смене операции очищать значения всех полей операции и возвращать select к первому варианту.
- Сохранять выбор товаров, фильтры, текущую вкладку и остальные данные, не относящиеся к форме операции.
- Добавить регрессионные JS-тесты для переходов между разными операциями и повторного выбора той же операции.

## Capabilities

### New Capabilities

Нет.

### Modified Capabilities

- `backend-ui-i18n`: уточняется поведение dependency-free JS при переключении операций в backend UI.

## Impact

- Код: `wa-apps/shop/plugins/masseditor/js/masseditor.js`.
- Тесты: `tests/js/masseditor.test.js`.
