## Why

Сейчас backend-список товаров показывает остатки по складам и позволяет менять остатки выбранного склада в массовой операции, но сам список нельзя сузить по конкретному складу. Из-за этого пользователю приходится вручную просматривать товары с остатками по нужному складу перед массовым изменением или выбором всех товаров по фильтру.

## What Changes

- Добавить в фильтры списка товаров поле `Склад` с вариантами `Все склады` и конкретные склады из `shop_stock`.
- Расширить серверную нормализацию фильтров значением `stock_id`, чтобы неизвестный или недоступный склад безопасно сбрасывался в `0`.
- Применять фильтр склада к странице товаров, поисковым подсказкам и серверному выбору всех товаров по текущему фильтру.
- Сохранять выбранный склад в ссылках пагинации и переходах между вкладками, где уже сохраняются остальные фильтры.
- Локализовать видимый текст фильтра через i18n-слой плагина.
- Не менять логику массовой операции `stock`: выбор склада для операции остается отдельным control и не подменяется фильтром списка.

## Capabilities

### New Capabilities

- Нет.

### Modified Capabilities

- `product-selection`: выборка товаров получает новый безопасный фильтр `stock_id`, применимый к списку, подсказкам и выбору всех товаров по фильтру.
- `backend-ui-i18n`: backend UI должен отобразить и сохранять фильтр `Склад` без хардкодных строк и без потери состояния при пагинации/переходах.

## Impact

- PHP: `shopMasseditorPluginProductSelectionService`, backend action и связанные helper-методы фильтров.
- Smarty: `templates/actions/backend/Backend.html` для control фильтра и URL-параметров.
- JS: `js/masseditor.js` только если требуется передать `stock_id` в подсказки или сброс/submit фильтра.
- Локализация: `.po`/i18n dictionary для label и варианта `Все склады`.
- Тесты: PHP-тесты фильтрации по складу, выбора всех по фильтру и подсказок; JS-тесты только при изменении `masseditor.js`.
- Документация для сверки перед реализацией:
  - https://developers.webasyst.ru/docs/cookbook/plugins/
  - https://developers.webasyst.ru/docs/cookbook/basics/classes/waModel/
  - https://developers.webasyst.ru/docs/cookbook/basics/classes/waRequest/

Non-goals:

- Не добавлять новые массовые операции и не менять правила применения операции `stock`.
- Не менять структуру таблиц плагина или таблиц Shop-Script.
- Не добавлять storefront/frontend-поведение: изменение относится только к backend UI плагина.
