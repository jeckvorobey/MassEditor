## Why

Массовая операция изменения остатков выбранного склада завершается общей ошибкой. После исправления маршрутизации операции SQL-запись в `shop_product_stocks` не передает обязательный для реальной схемы Shop-Script `product_id`, поэтому MariaDB откатывает транзакцию и пользователь не получает измененный остаток.

## What Changes

- Операция `stock` с выбранным складом (`stock_id > 0`) всегда записывает остаток в `shop_product_stocks` именно для выбранного склада.
- Если строки выбранного склада еще нет, старое значение считается `0`; операция создает строку выбранного склада и нулевые строки остальных настроенных складов, необходимые Shop-Script для корректного агрегата.
- Каждая создаваемая или обновляемая строка склада передает идентификатор товара, требуемый таблицей `shop_product_stocks`.
- Операция `stock` без выбранного склада (`stock_id = 0`) сохраняет прежнее поведение: меняет общий `shop_product_skus.count` только для товаров без складского учета и отклоняет товары со складскими строками.
- После складской записи пересчитываются агрегаты товара через механизм Shop-Script.
- UI, JS, локализация, схема БД и фильтр товаров не меняются.

## Capabilities

### New Capabilities

Нет.

### Modified Capabilities

- `mass-operations`: уточняются правила применения операции `stock` для выбранного склада и для режима без склада.

## Impact

- Код: `wa-apps/shop/plugins/masseditor/lib/classes/shopMasseditorPluginMassOperationService.class.php`.
- Тесты: `tests/php/MassOperationServiceTest.php`; проверка на Docker-стенде с реальной схемой Shop-Script.
- БД: безопасная пакетная запись в `shop_product_stocks` с полями `sku_id`, `stock_id`, `product_id`, `count`; обновление `shop_product.edit_datetime`, пересчет агрегатов `shop_product.count` и `shop_product_skus.count`.
- UI: клиентская валидация выбранного склада для выделенных товаров со складским учетом, с красной подсветкой селекта и существующим уведомлением об ошибке.
- Документация для сверки: хранение товаров Shop-Script, `waModel`, класс `shopProductStocksModel` из установленного Shop-Script.
