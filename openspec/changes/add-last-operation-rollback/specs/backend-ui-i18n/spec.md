## ADDED Requirements

### Requirement: Управление откатом в журнале
Вкладка `log` ДОЛЖНА (MUST) показывать локализованную кнопку отмены только у записи, которую backend признал доступной, и выполнять откат через отдельную CSRF-защищённую POST/AJAX-форму.

#### Scenario: Доступная запись журнала
- **WHEN** backend пометил строку `can_rollback=true`
- **THEN** Smarty ДОЛЖЕН вывести экранированный integer `log_id`, `{$wa->csrf()}` и кнопку из i18n-ключа `rollback_button`

#### Scenario: Недоступная запись журнала
- **WHEN** `can_rollback` отсутствует или false
- **THEN** строка MUST NOT содержать активную кнопку или клиентский способ отправить откат этой записи

#### Scenario: Размещение кнопки в журнале
- **WHEN** строка доступна для отката на desktop-ширине
- **THEN** компактная красная outline-кнопка с иконкой возврата ДОЛЖНА находиться внутри ячейки описания у правого края таблицы, а отдельная колонка действий MUST NOT создаваться

#### Scenario: Подтверждение
- **WHEN** пользователь нажимает кнопку отмены
- **THEN** dependency-free JS ДОЛЖЕН открыть modal с локализованным вопросом из `rollback_confirm_question` и кнопками подтверждения/отмены, не отправляя запрос до подтверждения

#### Scenario: Loader и защита от повтора
- **WHEN** пользователь подтверждает откат
- **THEN** JS ДОЛЖЕН заблокировать повторное подтверждение, показать доступный indeterminate loader, отправить FormData на `rollback_url` с same-origin credentials и скрыть loader только после результата

#### Scenario: Мобильные модальные и всплывающие окна
- **WHEN** любое модальное или всплывающее окно MassEditor открыто при ширине viewport не более `1024px`
- **THEN** общий modal card ДОЛЖЕН быть центрирован, занимать `80vw`, ограничиваться высотой экрана на узких устройствах, допускать вертикальную прокрутку и сохранять удобные полноширинные кнопки действий

#### Scenario: Безопасный результат
- **WHEN** сервер вернул успех или ошибку
- **THEN** JS ДОЛЖЕН показать локализованные заголовок/сообщение через `textContent`, а после успеха перезагрузить журнал для удаления кнопки

### Requirement: Двуязычные тексты отката
Все видимые тексты отката ДОЛЖНЫ (MUST) определяться в `shopMasseditorPluginI18nService`, включаться в JS i18n dictionary при необходимости и иметь переводы в `.po` каталогах `ru_RU` и `en_US`.

#### Scenario: Русский интерфейс
- **WHEN** `interface_language=ru_RU`
- **THEN** кнопка, вопрос «Вы действительно хотите вернуть старые значения?», loader, успех, конфликт и общая ошибка ДОЛЖНЫ отображаться по-русски

#### Scenario: Английский интерфейс
- **WHEN** `interface_language=en_US`
- **THEN** те же элементы ДОЛЖНЫ отображаться по-английски без русского fallback

#### Scenario: Экранирование и JS fallback
- **WHEN** UI показывает описание операции или серверную ошибку рядом с откатом
- **THEN** Smarty ДОЛЖЕН применять `escape`, JS ДОЛЖЕН использовать `textContent`, а hardcoded fallback в `masseditor.js` ДОЛЖЕН быть только английским
