## MODIFIED Requirements

### Requirement: UI видео и множественных характеристик
Backend UI MUST показывать только поля и режимы, допустимые для выбранной операции и типа характеристики, MUST заранее объяснять поддерживаемые источники видео и MUST передавать серверу нормализуемые значения без обхода общего подтверждения.

#### Scenario: Форма видео
- **WHEN** пользователь выбирает операцию `video`
- **THEN** UI ДОЛЖЕН показать режимы `set`/`clear`, поле URL только для `set` и локализованный placeholder с Rutube, VK, YouTube и Vimeo

#### Scenario: Неподдерживаемая видеоссылка
- **WHEN** пользователь вводит URL неподдерживаемого источника и пытается открыть confirm modal
- **THEN** UI ДОЛЖЕН оставить modal закрытым, показать красную рамку input, установить `aria-invalid=true` и вывести под полем локализованный текст о Rutube, VK, YouTube и Vimeo

#### Scenario: Сброс ошибки видео
- **WHEN** пользователь изменяет URL или выбирает режим `clear`
- **THEN** UI ДОЛЖЕН убрать красное состояние и inline-текст, а в режиме `clear` скрыть и отключить input

#### Scenario: Форма множественной характеристики
- **WHEN** пользователь выбирает характеристику с `multiple=1`
- **THEN** UI MUST показать режимы `replace`, `add`, `remove`, `clear` и multi-select существующих значений этой характеристики

#### Scenario: Очистка без значений
- **WHEN** выбран режим `clear` для видео или характеристики
- **THEN** UI MUST скрыть и отключить поле значения, чтобы устаревший ввод не попал в POST

#### Scenario: Подтверждение
- **WHEN** обязательные поля валидны и пользователь открывает modal
- **THEN** modal MUST показать локализованные название операции, режим и число товаров, а submit MUST добавить `confirm_apply=1`

### Requirement: Локализация релиза 1.2.0
Все новые видимые строки видео и множественных характеристик MUST поступать из `shopMasseditorPluginI18nService` и JS i18n dictionary и MUST иметь значения в `ru_RU` и `en_US`.

#### Scenario: Русский интерфейс
- **WHEN** `interface_language=ru_RU`
- **THEN** подписи режимов, URL, placeholder поддерживаемых видеосервисов, multi-select, подтверждение и ошибки MUST отображаться по-русски

#### Scenario: Английский интерфейс
- **WHEN** `interface_language=en_US`
- **THEN** подписи режимов, URL, placeholder поддерживаемых видеосервисов, multi-select, подтверждение и ошибки MUST отображаться по-английски без русского fallback

#### Scenario: Источник placeholder
- **WHEN** backend action формирует `texts` для Smarty-шаблона
- **THEN** placeholder видео ДОЛЖЕН поступать из i18n-ключа `video_url_placeholder`, а не быть захардкожен в шаблоне

#### Scenario: Экранирование
- **WHEN** UI отображает видеоссылку, placeholder, название характеристики, значение или серверную ошибку
- **THEN** Smarty/JS MUST использовать безопасный текстовый вывод без интерпретации пользовательского HTML
