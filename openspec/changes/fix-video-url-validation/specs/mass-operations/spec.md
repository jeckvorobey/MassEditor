## MODIFIED Requirements

### Requirement: Массовое редактирование видео
Операция `video` MUST безопасно устанавливать одну поддерживаемую Shop-Script ссылку всем выбранным товарам либо очищать существующую ссылку через документированный объект `shopProduct`.

#### Scenario: Установка поддерживаемой ссылки
- **WHEN** администратор подтверждает режим `set` с абсолютным HTTP(S) URL Rutube, VK, YouTube или Vimeo
- **THEN** сервис ДОЛЖЕН нормализовать URL через `shopVideo::checkVideo()`, проверить права на каждый товар, записать `video_url` через `shopProduct`, вызвать `save()` и подтвердить непустой результат

#### Scenario: Неподдерживаемый источник
- **WHEN** режим `set` содержит абсолютный HTTP(S) URL источника, который `shopVideo::checkVideo()` не поддерживает
- **THEN** сервис ДОЛЖЕН вернуть локализованную ошибку до транзакции, не изменять товары и не создавать успешный журнал

#### Scenario: Очистка ссылки
- **WHEN** администратор подтверждает режим `clear`
- **THEN** сервис ДОЛЖЕН сохранить пустой `video_url` через `shopProduct`, вызвать `save()` и подтвердить отсутствие URL

#### Scenario: Неверная ссылка
- **WHEN** режим `set` содержит пустую, относительную, не-HTTP(S) или слишком длинную ссылку
- **THEN** сервис ДОЛЖЕН вернуть локализованную ошибку до первой записи

#### Scenario: Ошибка сохранения товара
- **WHEN** `save()` возвращает неуспех, итоговое состояние не соответствует режиму или сохранение любого товара завершается исключением
- **THEN** сервис ДОЛЖЕН выполнить `ROLLBACK`, не записывать успешный журнал и не продолжать операцию как успешную

#### Scenario: Подтверждение и журнал
- **WHEN** массовая операция видео успешно завершена
- **THEN** она MUST пройти общий confirm-flow и создать запись `video` в `shop_masseditor_log`
