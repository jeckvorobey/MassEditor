## ADDED Requirements

### Requirement: Согласованный релизный документ 1.2.0
Документ `docs/release-1.2.0.md` ДОЛЖЕН (MUST) соответствовать фактическому пользовательскому scope версии `1.2.0` и согласовываться с RU/EN changelog и Store-описанием.

#### Scenario: Полный пользовательский scope после 1.1.0
- **WHEN** релизный документ подготовлен к публикационной сборке
- **THEN** он ДОЛЖЕН перечислять фильтр типа складского учёта, множественные общие характеристики, массовое видео с проверкой поддерживаемых источников, уточнённые подписи, progress/result modal, сброс формы после успеха и ограниченный откат последней операции
- **AND** он НЕ ДОЛЖЕН обещать SKU- или дочерние характеристики, создание новых справочных значений, загрузку видеофайлов, отдельную ссылку для каждого товара, точный процент прогресса или откат произвольной операции

#### Scenario: Исторические версии остаются неизменными
- **WHEN** материалы `1.2.0` регенерируются из release dataset
- **THEN** отдельные документы и changelog-блоки `1.0.0` и `1.1.0` НЕ ДОЛЖНЫ изменяться
- **AND** существующий блок `1.2.0` ДОЛЖЕН обновляться на месте без дублирования

#### Scenario: Границы ручной проверки
- **WHEN** скриншоты и ручное UI-тестирование выполняются отдельно пользователем
- **THEN** release-документ ДОЛЖЕН сохранять команды автоматической проверки и НЕ ДОЛЖЕН утверждать, что внешняя Store-публикация уже выполнена
