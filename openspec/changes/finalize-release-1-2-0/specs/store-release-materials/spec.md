## ADDED Requirements

### Requirement: Согласованный релизный документ 1.2.0
Документ `docs/release-1.2.0.md` ДОЛЖЕН (MUST) соответствовать фактическому пользовательскому scope версии `1.2.0` и согласовываться с RU/EN changelog и Store-описанием.

#### Scenario: Основные функции версии 1.2.0
- **WHEN** релизный документ подготовлен к публикационной сборке
- **THEN** он ДОЛЖЕН перечислять фильтр типа складского учёта, массовое видео и множественные общие характеристики с режимами замены, добавления, удаления и очистки
- **AND** он НЕ ДОЛЖЕН обещать SKU- или дочерние характеристики, создание новых справочных значений, загрузку видеофайлов или отдельную ссылку для каждого товара

#### Scenario: Границы ручной проверки
- **WHEN** скриншоты и ручное UI-тестирование выполняются отдельно пользователем
- **THEN** release-документ ДОЛЖЕН сохранять команды автоматической проверки и НЕ ДОЛЖЕН утверждать, что внешняя Store-публикация уже выполнена
