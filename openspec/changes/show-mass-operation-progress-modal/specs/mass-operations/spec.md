## ADDED Requirements

### Requirement: Наблюдаемое выполнение массовой операции
После явного подтверждения backend UI MUST отправлять операцию в защищённый JSON controller и показывать отдельное модальное состояние выполнения до получения итогового ответа. Controller ДОЛЖЕН сохранить проверку администратора, CSRF, серверную валидацию, лимит, транзакцию, rollback и журналирование существующего сервиса.

#### Scenario: Операция выполняется
- **WHEN** пользователь подтверждает валидную массовую операцию и запрос отправлен
- **THEN** окно подтверждения ДОЛЖНО закрыться, окно выполнения ДОЛЖНО показать неопределённый индикатор, а повторная отправка и закрытие ДОЛЖНЫ быть недоступны

#### Scenario: Операция завершилась успешно
- **WHEN** JSON controller возвращает успешный результат
- **THEN** окно выполнения ДОЛЖНО показать итоговое сообщение, остаться открытым до ручного закрытия и только после закрытия ДОЛЖНО очистить выбор товаров и перезагрузить страницу с дефолтной формой и обновлённой таблицей

#### Scenario: Операция завершилась ошибкой
- **WHEN** controller возвращает ошибку валидации, выполнения или сети
- **THEN** окно выполнения ДОЛЖНО показать безопасное сообщение, остаться открытым до ручного закрытия и после закрытия ДОЛЖНО сохранить заполненную форму и выбранные товары для исправления

#### Scenario: Fallback без Fetch API
- **WHEN** браузер не поддерживает `fetch` или `FormData`
- **THEN** форма ДОЛЖНА сохранить существующий защищённый обычный POST flow
