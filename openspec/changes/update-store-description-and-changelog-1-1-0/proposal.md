## Why

Описание Mass Editor для Webasyst Store и существующие релизные материалы версии `1.1.0` больше не соответствуют фактическим возможностям плагина: они ограничивают редактирование характеристик базовыми типами и не отражают новые складские, мобильные и интерфейсные улучшения. Нужен компактный двуязычный источник правды для карточки продукта и истории выпусков, чтобы текущий релиз был описан без устаревших ограничений, а будущие версии получали синхронные RU/EN записи changelog.

## What Changes

- Обновить RU/EN блоки в `docs/store-description-1_1_0.md`: добавить складской фильтр и остаток выбранного склада, расширенные поддерживаемые типы характеристик и мобильный выбор всех товаров текущей страницы.
- Уточнить ограничения характеристик: не обещать SKU-, дочерние и множественные характеристики, а также создание произвольных новых значений.
- Создать или восстановить синхронную пару `docs/CHANGELOG.md` и `docs/CHANGELOG.en.md` как постоянную историю версий продукта.
- Добавить компактную запись `1.1.0 — 2026-07-10` с пользовательскими изменениями относительно `1.0.0`, включая исправление применения складских остатков и отображения пункта плагина в меню админ-панели Shop-Script; копируемое содержимое оформить Store-ready HTML-блоком, а не Markdown-списком.
- Привести `docs/release-1.1.0.md` и `docs/store-publication.md` в соответствие с актуальным набором поддерживаемых характеристик и функций.
- Сохранить Store-ready HTML только на базовых разрешенных тегах, без CSS, скриптов и неподтвержденных маркетинговых обещаний; обеспечить фактическую эквивалентность русского и английского текста.
- Проверить материалы по официальным требованиям Webasyst к публикации, подготовке обновлений и оформлению карточки продукта.
- Границы задачи: runtime-код, версия в `plugin.php`, локализация интерфейса, изображения и скриншоты не изменяются.

## Capabilities

### New Capabilities

- `store-release-materials`: двуязычное описание плагина для Webasyst Store, синхронная RU/EN история версий и правила подготовки компактных релизных записей на основе фактически реализованных функций.

### Modified Capabilities

Нет.

## Impact

- Документация: `docs/store-description-1_1_0.md`, `docs/CHANGELOG.md`, `docs/CHANGELOG.en.md`, `docs/release-1.1.0.md`, `docs/store-publication.md`.
- OpenSpec: новая capability-spec `store-release-materials` и артефакты этого change.
- Пользовательский эффект: карточка Webasyst Store будет точно описывать возможности `1.1.0`, а раздел обновлений получит короткую двуязычную запись, пригодную для последующего ведения по версиям.
- Проверки: сверка RU/EN фактов, проверка разрешенных HTML-тегов, поиск устаревших ограничений, `openspec validate --strict --all`.
- Официальные источники для реализации:
  - https://developers.webasyst.ru/docs/store/product-update/
  - https://developers.webasyst.ru/docs/store/webasyst-store-requirements/
  - https://developers.webasyst.ru/docs/store/better-product-sales/
- Runtime, API, схема БД, зависимости, права доступа и поведение backend UI не затрагиваются; TDD для кода не требуется, поскольку изменение документационное.
