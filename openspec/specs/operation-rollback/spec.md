## Purpose

Зафиксировать безопасный ограниченный откат последней массовой операции над товарами без раскрытия сохранённых значений и перезаписи последующих изменений.

## Requirements

### Requirement: Ограниченная доступность отката
Плагин ДОЛЖЕН (MUST) разрешать отмену только глобально последней успешной массовой операции текущего backend-пользователя в течение 3 часов после её завершения.

#### Scenario: Последняя операция доступна
- **WHEN** текущий пользователь открыл журнал, его операция имеет максимальный `shop_masseditor_log.id`, снимок не отменён и `expires_at` ещё не наступил
- **THEN** backend ДОЛЖЕН пометить только эту запись как доступную для отката

#### Scenario: Появилась следующая операция
- **WHEN** после исходной операции успешно записана любая следующая операция MassEditor
- **THEN** исходная запись ДОЛЖНА перестать быть доступной для отката независимо от оставшегося времени

#### Scenario: Истекли 3 часа
- **WHEN** серверное время позже `expires_at` исходного снимка
- **THEN** кнопка ДОЛЖНА отсутствовать, а прямой запрос отката ДОЛЖЕН быть отклонён без записи

#### Scenario: Другой пользователь
- **WHEN** текущий backend-пользователь не совпадает с `user_id` исходной операции и снимка
- **THEN** UI ДОЛЖЕН скрыть кнопку, а controller/service ДОЛЖНЫ отклонить прямой запрос

#### Scenario: Повторный откат
- **WHEN** снимок уже содержит `rolled_back_at`
- **THEN** повторный запрос ДОЛЖЕН быть отклонён без изменения товаров и нового успешного аудита

### Requirement: Безопасный запрос отката
Откат ДОЛЖЕН (MUST) выполняться только через POST/AJAX после явного подтверждения, проверки CSRF, прав администратора Shop-Script, положительного integer `log_id` и прав редактирования каждого товара снимка.

#### Scenario: Нет подтверждения
- **WHEN** POST не содержит `confirm_rollback=1`
- **THEN** controller/service ДОЛЖНЫ отклонить запрос до первой записи

#### Scenario: Нет прав администратора
- **WHEN** пользователь не является администратором `shop`
- **THEN** controller и service ДОЛЖНЫ вернуть безопасную локализованную ошибку и не читать содержимое снимка в response

#### Scenario: Нет прав на товар
- **WHEN** `shopProduct::checkRights()` возвращает false хотя бы для одного товара снимка
- **THEN** весь откат ДОЛЖЕН быть отклонён до первой записи

#### Scenario: Невалидный ID
- **WHEN** `log_id` равен `0`, отрицательный, дробный, строковый SQL/HTML payload или не соответствует снимку
- **THEN** запрос ДОЛЖЕН быть отклонён через integer validation/placeholders без интерпретации payload

### Requirement: Снимок только затронутых данных
Плагин ДОЛЖЕН (MUST) сохранять до и после операции версионированное состояние только тех whitelisted полей и связей, которые может изменить подтверждённый action type, отдельно по каждому товару и без N+1 чтений.

#### Scenario: Операция над товаром или SKU
- **WHEN** применяется visibility, description, url, video, price, compare_price, availability или stock
- **THEN** снимок ДОЛЖЕН содержать только соответствующие поля товара/SKU/складов и необходимые агрегаты, загруженные батчево

#### Scenario: Теги, категории или характеристика
- **WHEN** применяется tags, categories или features
- **THEN** снимок ДОЛЖЕН содержать только теги товара, категории товара либо product-level связи подтверждённого `feature_id` и MUST NOT включать другие характеристики или SKU-feature rows

#### Scenario: Снимок недопустимого размера или формата
- **WHEN** число items превышает operation limit, JSON повреждён, версия/тип/ID не соответствуют заголовку или item превышает допустимый размер
- **THEN** операция или откат ДОЛЖНЫ завершиться отказом без сохранения частичного снимка и без раскрытия его содержимого

### Requirement: Защита от перезаписи следующего состояния
Перед восстановлением сервис ДОЛЖЕН (MUST) повторно прочитать затронутые поля и сравнить их с `after_state`; восстановление разрешено только при точном нормализованном совпадении.

#### Scenario: Состояние не менялось
- **WHEN** текущие значения всех затронутых полей совпадают с `after_state`
- **THEN** сервис ДОЛЖЕН перейти к восстановлению `before_state`

#### Scenario: Конфликт после операции
- **WHEN** хотя бы одно затронутое значение товара, SKU, склада или связи отличается от `after_state`
- **THEN** сервис ДОЛЖЕН отклонить весь откат до первой записи и показать безопасное локализованное сообщение о конфликте

#### Scenario: Параллельные запросы MassEditor
- **WHEN** применение или откат уже удерживает общую блокировку MassEditor
- **THEN** второй mutating request ДОЛЖЕН дождаться ограниченное время либо завершиться безопасной ошибкой, а eligibility ДОЛЖНА проверяться после получения блокировки

### Requirement: Точное и компенсируемое восстановление
Откат ДОЛЖЕН (MUST) восстановить только `before_state` исходной операции ограниченными батчами, пересчитать требуемые Shop-Script агрегаты и вернуть систему к `after_state`, если восстановление завершилось исключением после частичной записи.

#### Scenario: Успешное восстановление
- **WHEN** eligibility, права и `after_state` подтверждены
- **THEN** сервис ДОЛЖЕН восстановить старые значения всех товаров снимка и MUST NOT менять несвязанные поля/связи

#### Scenario: Ошибка в середине отката
- **WHEN** после части записей восстановления возникает исключение
- **THEN** сервис ДОЛЖЕН компенсирующе вернуть `after_state`, не помечать снимок отменённым и не писать успешный rollback audit

#### Scenario: Остатки
- **WHEN** отменяется stock-операция
- **THEN** сервис ДОЛЖЕН восстановить существовавшие значения, удалить только созданные исходной операцией складские строки и вызвать штатный пересчёт агрегатов затронутых товаров

### Requirement: Аудит без раскрытия старых значений
После полного успешного восстановления плагин ДОЛЖЕН (MUST) пометить снимок отменённым и записать новую строку `rollback` в `shop_masseditor_log` с ID исходной операции, пользователем и количеством товаров, но без `before_state`/`after_state`.

#### Scenario: Успешный аудит
- **WHEN** все товары восстановлены и снимок условно помечен `rolled_back_at`
- **THEN** журнал ДОЛЖЕН получить одну запись `rollback`, а исходная операция ДОЛЖНА стать недоступной для повторной отмены

#### Scenario: Техническая ошибка
- **WHEN** откат завершается неожиданным исключением
- **THEN** JSON response ДОЛЖЕН содержать только локализованную общую ошибку, а класс/сообщение/trace ДОЛЖНЫ записываться только в `shop/plugins/masseditor.log`
