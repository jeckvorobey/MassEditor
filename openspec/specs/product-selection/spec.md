## Purpose

Зафиксировать контракт выборки товаров, фильтров, пагинации, подсказок поиска и пакетной загрузки связанных данных.

## Requirements

### Requirement: Серверная выборка товаров
Список товаров MUST формироваться серверно через `shopMasseditorPluginProductSelectionService` и `shopMasseditorPluginProductModel`, с нормализацией фильтров и безопасными параметрами SQL.

#### Scenario: Получение страницы товаров
- **WHEN** backend-экран запрашивает список товаров
- **THEN** сервис MUST вернуть `products`, нормализованные `filters` и `pagination` с корректными `page`, `page_size`, `total`, `pages`

#### Scenario: Некорректные фильтры
- **WHEN** входные фильтры содержат неизвестный `status` или `availability`
- **THEN** сервис MUST привести их к `all`

### Requirement: Расширенный поиск
Поиск MUST поддерживать название, основной SKU, URL, summary, description, теги и категории без прямой склейки пользовательского ввода в SQL.

#### Scenario: Поисковый запрос
- **WHEN** пользователь вводит строку поиска
- **THEN** SQL MUST использовать placeholder-параметры, а связанные таблицы SKU, тегов и категорий MUST подключаться только через сервис выборки

#### Scenario: Поисковые подсказки
- **WHEN** query короче 2 символов
- **THEN** controller подсказок MUST вернуть пустой список без запроса к сервису

#### Scenario: Подсказки с активными фильтрами
- **WHEN** запрашиваются подсказки при активных status, availability или category фильтрах
- **THEN** сервис MUST применить эти фильтры, игнорируя текущий query фильтр страницы, и вернуть уникальные ограниченные значения

### Requirement: Пагинация и лимиты
Пагинация MUST нормализовать номер страницы и размер страницы, чтобы не выходить за доступный диапазон и не делать чрезмерные выборки.

#### Scenario: Page size вне диапазона
- **WHEN** page size не задан или превышает верхнюю границу
- **THEN** сервис MUST использовать безопасное значение: по умолчанию 50, максимум 200

### Requirement: Выбор всех товаров по фильтру
Выбор всех найденных товаров MUST выполняться серверной повторной выборкой ID по текущим фильтрам с учетом лимита операции.

#### Scenario: Filter selection
- **WHEN** пользователь выбирает все товары по текущему фильтру
- **THEN** `getIdsByFilters()` MUST вернуть distinct ID в серверном порядке и ограничить выборку лимитом операции

### Requirement: Пакетная загрузка связанных данных
Список товаров MUST загружать категории, теги, SKU и остатки батчами, без N+1 запросов по каждому товару.

#### Scenario: Остатки по складам
- **WHEN** страница содержит товары и склады
- **THEN** `attachStockDetails()` MUST одним батчем загрузить остатки по product IDs, сохранить общий `count` товара и показать `∞`, если хотя бы один складской остаток SKU бесконечный
