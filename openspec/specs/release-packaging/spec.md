## Purpose

Зафиксировать требования к воспроизводимой и безопасной сборке релизного архива плагина MassEditor для Webasyst Store.

## Requirements

### Requirement: Источник и имя релизного архива
Система сборки MUST определять product ID, slug и трёхчастную версию из конфигурации целевого Webasyst-продукта, создавать архив штатной командой `php wa.php compress <slug>` и помещать проверенный артефакт в согласованный project path с именем, содержащим фактическую версию.

#### Scenario: Сборка текущей версии
- **WHEN** конфигурация содержит числовой `vendor`, валидную версию и доступный Webasyst CLI
- **THEN** workflow ДОЛЖЕН запустить `php wa.php compress <slug>` без `-skip test`, определить созданный архив по фактическому результату и проверить совпадение версии конфигурации, release manifest, документов и имени артефакта

#### Scenario: Некорректная версия, vendor или недоступный CLI
- **WHEN** версия не состоит из трёх чисел, `vendor` не является числовым либо штатный CLI недоступен
- **THEN** сборка ДОЛЖНА завершиться понятным blocker до замены существующего архива
- **AND** самодельный fallback MUST NOT использоваться без явного разрешения

### Requirement: Формат архива Webasyst Store
Релизный артефакт MUST (ДОЛЖЕН) быть корректным `tar.gz` архивом с единственной корневой директорией `masseditor/` и безопасными относительными путями.

#### Scenario: Проверка корневой структуры
- **WHEN** читается manifest собранного архива
- **THEN** каждый путь ДОЛЖЕН начинаться с `masseditor/`
- **AND** архив НЕ ДОЛЖЕН содержать второй верхнеуровневый файл или каталог

#### Scenario: Проверка безопасных путей
- **WHEN** проверяется manifest архива
- **THEN** он НЕ ДОЛЖЕН содержать абсолютные пути, переходы `..`, символические ссылки за пределы корня или специальные файлы

### Requirement: Только production-файлы плагина
Архив MUST содержать ожидаемый distribution manifest продукта и MUST NOT содержать секреты, логи, дампы, временные и лишние development-файлы. Проверка MUST учитывать допустимые исключения официального упаковщика и MUST NOT требовать безусловного нулевого diff со всем source tree.

#### Scenario: Проверка distribution manifest
- **WHEN** кандидат распакован во временную директорию
- **THEN** его содержимое ДОЛЖНО совпасть с предварительно зафиксированным distribution manifest обязательных и разрешённых файлов

#### Scenario: Запрещённый файл
- **WHEN** manifest содержит `.git`, `.env`, ключ, дамп, лог, временный файл, абсолютный путь, `..`, небезопасную ссылку или второй корневой каталог
- **THEN** кандидат ДОЛЖЕН быть отклонён и существующий архив MUST NOT измениться

### Requirement: Защищенные директории и публикационные метаданные
Архив MUST (ДОЛЖЕН) сохранять Webasyst-структуру плагина, обязательные защитные файлы и декларацию совместимости Shop-Script.

#### Scenario: Проверка защищенных директорий
- **WHEN** проверяются `masseditor/lib/`, `masseditor/templates/` и `masseditor/locale/` в распакованном архиве
- **THEN** каждая корневая защищенная директория ДОЛЖНА содержать `.htaccess` с единственной директивой `Deny from all`

#### Scenario: Проверка конфигурации Shop-Script
- **WHEN** проверяются файлы `lib/config/plugin.php` и `lib/config/shop_support.json`
- **THEN** конфигурация ДОЛЖНА содержать ID-контекст `masseditor`, числовой `vendor`, версию `1.2.0` и явную декларацию поддержки или отсутствия поддержки PREMIUM-возможностей

### Requirement: Актуальные локализации
Релизный архив MUST (ДОЛЖЕН) содержать проверенные пары `.po` и `.mo` для `ru_RU` и `en_US`, собранные из текущих исходников переводов до упаковки.

#### Scenario: Компиляция обеих локалей
- **WHEN** выполняется релизная сборка
- **THEN** каждый `shop_masseditor.po` ДОЛЖЕН успешно пройти gettext-проверку и сформировать соответствующий `shop_masseditor.mo`

#### Scenario: Инструмент локализации недоступен
- **WHEN** проверенный gettext-совместимый компилятор недоступен или сообщает ошибку
- **THEN** сборка ДОЛЖНА остановиться и НЕ ДОЛЖНА заменять существующий архив

### Requirement: Релизный quality gate
Финальный архив MUST заменять предыдущий draft-файл только после успешных project tests, version/localization/changelog/meta-update checks, `gzip`/`tar`/manifest validation и secret scan. Архив версии со статусом `published` MUST NOT перезаписываться без отдельного явного разрешения.

#### Scenario: Успешная draft-сборка
- **WHEN** все обязательные автоматические проверки завершились успешно и версия имеет статус `draft`
- **THEN** кандидат ДОЛЖЕН атомарно заменить draft-артефакт
- **AND** итоговый report ДОЛЖЕН содержать путь, размер, число файлов, SHA-256, команды и оставшиеся ручные действия

#### Scenario: Попытка перезаписать published-архив
- **WHEN** release manifest помечает существующую версию как `published` и отдельное force-разрешение отсутствует
- **THEN** workflow ДОЛЖЕН заблокировать замену до создания временного финального файла

#### Scenario: Ошибка обязательной проверки
- **WHEN** любая обязательная проверка кандидата завершается ошибкой
- **THEN** существующий финальный архив MUST NOT быть заменён
- **AND** статус `READY` MUST NOT быть выдан
