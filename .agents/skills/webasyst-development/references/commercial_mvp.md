# Commercial Webasyst MVP boundaries

Load this reference while shaping a new commercial Webasyst or Shop-Script product, or when a requested change may widen an MVP.

Describe the product through one merchant pain:

```text
Для кого: конкретная роль или тип магазина
Боль: повторяющаяся проблема
Решение: одна главная операция
Результат: измеримый выигрыш
```

Keep the first release to one primary scenario and one clear backend flow. Prefer backend-only delivery unless a customer-facing page, SEO, or a public route is essential to the value.

Include preview, explicit confirmation, bounded writes, and an operation log for destructive or bulk operations. Exclude universal workflow builders, speculative settings, unrelated integrations, unnecessary storefront surfaces, and broad entity support.

For each release record the user pain, included scope, explicit non-goals, verification path, and Store publication risks. Split independent user scenarios into separate releases or products.
