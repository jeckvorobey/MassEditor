# Массовый редактор 1.2.0

## Что добавлено

- **Фильтр типа складского учёта**: при массовой операции «Остатки» без выбора склада (`stock_id = 0`) появился фильтр с тремя вариантами: «Все товары» (по умолчанию), «Без складского учёта», «Со складским учётом». Фильтр позволяет безопасно применять массовое изменение остатков только к нужной категории товаров, избегая рассинхронизации данных для товаров со складским учётом.

## Что исправлено (из 1.1.1)

- **Остатки для товаров без складского учёта**: при массовой операции «Остатки» с выбором склада, товары без складского учёта больше не вызывают ошибку. Плагин корректно обновляет количество через `sku.count` вместо записи в `shop_product_stocks`.
- **Пункт в расширенном меню**: «Массовый редактор» теперь отображается в навигации бэкенда Shop-Script через `backend_extended_menu` с иконкой редактирования.
- **Docker-стенд**: порт изменён на 8088, чтобы не конфликтовать с другими локальными сервисами.

## Что улучшено

- Название плагина унифицировано на «Массовый редактор» во всех документах и описании Store.
- Добавлен английский текст обновления для двуязычной публикации.
- Рефакторинг `backendMenu()` — URL генерируется через общий метод `getBackendUrl()`.

## Почему версия 1.2.0

- `MAJOR` не меняется — обратная совместимость сохранена.
- `MINOR` увеличивается — добавлена новая функциональность (фильтр типа учёта).
- `PATCH` сбрасывается на 0.
