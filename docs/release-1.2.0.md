# Массовый редактор 1.2.0

## Что добавлено

- **Фильтр типа складского учёта**: при массовой операции «Остатки» без выбора склада (`stock_id = 0`) появился фильтр с тремя вариантами: «Все товары» (по умолчанию), «Без складского учёта», «Со складским учётом». Фильтр позволяет безопасно применять массовое изменение остатков только к нужной категории товаров, избегая рассинхронизации данных для товаров со складским учётом.
- **Множественные общие характеристики**: для характеристик с несколькими значениями доступны замена набора, добавление, удаление только выбранных значений и полная очистка. Перед сохранением значения проверяются и должны принадлежать выбранной характеристике.
- **Массовое видео**: выбранным товарам можно установить одну HTTP(S)-ссылку на видео или очистить текущую ссылку.
- **Штатное обновление установленного плагина**: схема журнала безопасно доводится идемпотентным meta-update Webasyst без собственной таблицы миграций.

## Границы версии

- Поддерживаются только общие характеристики товара; SKU-характеристики и дочерние характеристики не изменяются.
- Для списков и множественных характеристик используются существующие значения; создание новых справочных значений не входит в релиз.
- Для видео одна ссылка применяется ко всем выбранным товарам; загрузка видеофайлов и разные ссылки для каждого товара не поддерживаются.

## Что исправлено (из 1.1.1)

- **Остатки для товаров без складского учёта**: при массовой операции «Остатки» с выбором склада, товары без складского учёта больше не вызывают ошибку. Плагин корректно обновляет количество через `sku.count` вместо записи в `shop_product_stocks`.
- **Пункт в расширенном меню**: «Массовый редактор» теперь отображается в навигации бэкенда Shop-Script через `backend_extended_menu` с иконкой редактирования.
- **Docker-стенд**: порт изменён на 8088, чтобы не конфликтовать с другими локальными сервисами.

## Что улучшено

- Название плагина унифицировано на «Массовый редактор» во всех документах и описании Store.
- Добавлен английский текст обновления для двуязычной публикации.
- Рефакторинг `backendMenu()` — URL генерируется через общий метод `getBackendUrl()`.

## Почему версия 1.2.0

- `MAJOR` не меняется — обратная совместимость сохранена.
- `MINOR` увеличивается — добавлены фильтр типа складского учёта, множественные общие характеристики и массовое видео.
- `PATCH` сбрасывается на 0.

## Проверка перед публикацией

```bash
bash tests/run-php-tests.sh
bash tests/run-js-tests.sh
find wa-apps/shop/plugins/masseditor/lib -type f -name '*.php' -print0 | xargs -0 -n1 php -l
node --check wa-apps/shop/plugins/masseditor/js/masseditor.js
gzip -t dist/masseditor-1.2.0.tar.gz
tar -tzf dist/masseditor-1.2.0.tar.gz
```

Перед загрузкой архив нужно распаковать и подтвердить нулевой рекурсивный diff относительно `wa-apps/shop/plugins/masseditor/`. Скриншоты и итоговую ручную проверку интерфейса выполняют отдельно; наличие этого документа не означает, что архив уже загружен или опубликован в Webasyst Store.
